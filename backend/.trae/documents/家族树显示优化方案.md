# 家族树显示优化方案 - 确保完整显示姓名和性别

## 问题分析

### 当前问题
1. **字体增大后仍可能显示不全** - 如"欧阳大成（男）"可能被省略
2. **需要确保"XX（男/女）"完整显示** - 不能出现省略号
3. **人名和性别之间不要加空格** - 节省空间

### 当前显示逻辑
**文件**: `miniprogram/pages/tree/tree.js` 第 669-673 行

```javascript
getDisplayName(member) {
  const name = member && member.name ? member.name : '未命名成员'
  const gender = this.getGenderTag(member)
  return gender ? `${name}（${gender}）` : name  // ✅ 已经没空格了
}
```

**当前格式**: `欧阳大成（男）` - 姓名 + 中文括号 + 性别

## 解决方案

### 方案 1: 调整卡片宽度（推荐）

**思路**: 在卡片大小不变的情况下，优化内部布局

#### 修改 1: 减小 padding
**文件**: `miniprogram/pages/tree/tree.wxss` 第 599-608 行

```css
/* 修改前 */
.tree-node {
  padding: 10rpx 12rpx 10rpx;
}

/* 修改后 */
.tree-node {
  padding: 8rpx 10rpx 8rpx;  /* 减小 2rpx */
}
```

**效果**: 增加内部可用空间约 4rpx

#### 修改 3: 优化省略策略
**文件**: `miniprogram/pages/tree/tree.js`

**问题**: `displayName` 包含姓名和性别，如果太长会被省略

**解决**: 确保性别部分不被省略，只省略超长的姓名

**修改 `getDisplayName` 方法**:

```javascript
getDisplayName(member) {
  const name = member && member.name ? member.name : '未命名成员'
  const gender = this.getGenderTag(member)
  
  // 短名字直接返回完整格式
  if (name.length <= 4) {
    return gender ? `${name}（${gender}）` : name
  }
  
  // 长名字：缩短名字但保留性别
  // 如"欧阳大成大伟" → "欧阳大...（男）"
  if (gender && name.length > 6) {
    const shortName = name.substring(0, 3) + '...'
    return `${shortName}（${gender}）`
  }
  
  return gender ? `${name}（${gender}）` : name
}
```

**优点**:
- ✅ 确保性别始终显示
- ✅ 避免"欧阳大成（男）"被截断成"欧阳大成（..."
- ✅ 保持格式统一
